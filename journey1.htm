Our Journey

Connecting the Data

What Worked

The building footprints were successfully uploaded onto MapBox.

What Didn't Work

The method in loading the building footprints as a GeoJSON file onto a server and linking the file using MapBox JS did not work. Instead, we uploaded our ODB data as tilesets and then referenced the ODB layer in the map.addLayer function.

What We Had to Deviate From

The building footprints are uploaded directly onto MapBox instead of being hosted on a server.



Displaying the Data

What Worked

The building footprints are successfully displayed on the final product. The building polygons can be styled by the user, which was implemented by following this tutorial.

What Didn't Work

The dissemination area and dissemination blocks were unable to be styled on the map (link to page). If the areas and blocks were included, we wanted to display it as a toggle-able layer similar to the one in this tutorial.

What We Had to Deviate From

The dissemination area and block polygons are omitted from the final product.



Adding a Search Bar

What Worked

The geocoding tutorial from MapBox was used to add a search bar to the web page. This tutorial makes use of the Mapbox Geocoding API which transforms written addresses and building names into latitude and longitude coordinates


What Didn't Work

The search bar was unable to search and display information other than addresses.

What We Had to Deviate From

The initial brainstorm of the project intended for the search bar to be able to look up addresses and search by sqm2. The idea changed to incorporate more geographic functions offered by MapBox. These ideas included adding a local search restriction to limit the search results to the area displayed on the screen, and bounding box to limit results to the study area of Fredericton by coordinates.



Zoom Levels

What Worked

The map navigation controls for displaying zoom in and zoom out buttons were able to be displayed properly on screen using this tutorial. Highlighting over the zoom in and zoom out buttons brings up a text bubble which displays what each buttons function is.

What Didn't Work

Customized zoom levels were unable to be displayed at a dissemination area and dissemination block level because it required specific coordinates defining the view by the fit bounds method we were using. This may require further research to implement in the future.

What We Had to Deviate From

The bounding box feature was used as a substitute to display the study area at different zoom levels instead, following this tutorial. This feature comes in the form of a clickable button that sends the user to 3 pre-set areas; New Brunswick, Fredericton, and building level.



Pop-Ups

What Worked

When hovered over the building footprint polygons, a pop up shows the area in sqm2. This was successful following this tutorial.

What Didn't Work

The pop up code from this example did not work during testing, even when the code was copied over.

What We Had to Deviate From

There was no deviation from the original plan, as the pop ups worked and displayed the intended information.
